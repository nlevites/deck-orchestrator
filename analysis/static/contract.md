## 5. Notes

- The executor uses *short-poll* on `GET /executor/poll` (returns 204 immediately when no work is ready). At default 500ms poll interval and 0 % occupancy, every poll is a wasted byte. See `inefficiencies/inefficiencies.md` S1.
- `/api/state` ships the full `decks[]` slice on **every** client poll (~30 B/deck/Hz). See README.md in `frontend/src/lib/live/`. Captured in inefficiency S3.
- Error envelopes for `VERSION_MISMATCH` / `ALREADY_TERMINAL` embed the full `current_state: Run` — see inefficiency S5.
