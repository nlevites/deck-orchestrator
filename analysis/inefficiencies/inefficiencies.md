## Details

### C1 — Console keeps polling while tab is hidden _(high)_

**Summary.** `use-live-state.ts` does not gate the poll loop on `document.visibilityState`.

- Static scan: no `visibilityState` reference in `frontend/src/lib/live/use-live-state.ts`.
- Impact: a backgrounded operator tab keeps issuing 1 req/s to `/api/state`. 
  At an ops shift with 5 stale tabs forgotten on a workstation, that's 5 req/s 
  of pure noise the orchestrator commits to the read pool.
- Scenarios with visibility transitions recorded: 0.

**Mitigation.** Gate the poll: wrap `tick()` with `if (document.visibilityState === "hidden") return;` and add a `visibilitychange` listener that fires an immediate catch-up tick when the tab comes back.

**Backing metrics:**
```json
{
  "gated_on_visibility": false,
  "scenarios_with_visibility_change": 0
}
```

### C2 — Multi-tab cost is linear in tab count _(high)_

**Summary.** multi-tab/steady-state bytes ratio = 3.0× (3 tabs ≈ 3×).

- steady-state (1 tab) bytes: 5201.7 KiB
- multi-tab (3 tabs) bytes: 15700.1 KiB
- ratio ~= number of tabs → no coalescing.

**Mitigation.** Use a `SharedWorker` or `BroadcastChannel` so one tab drives the poll loop and broadcasts state deltas to siblings. Cuts orchestrator request rate by N-1/N.

**Backing metrics:**
```json

  "bytes_multi": 16076856,
  "bytes_steady": 5326543,
  "tab_multiplier": 3.0182533023764195
}
```

### S1 — Wasted executor polls (204 No Content rate) _(high)_

**Summary.** 87% of /executor/poll returned 204 — pure-overhead requests.

- N=117 poll requests in capture, 102 returned 204 (87.2%).
- At 0% occupancy every poll is a wasted byte. Defaults: poll every 500ms => 2 wasted req/s/deck => 200 req/s at 100 decks just to ask `anything for me?`

**Mitigation.** Long-poll the executor (block up to N seconds inside the handler) or push via SSE/server-sent events. Both shrink idle traffic by ~99%.

**Backing metrics:**
```json
{
  "n_polls": 117,
  "n_204": 102,
  "pct_204": 87.17948717948718
}
```

### S2 — Header-to-body ratio on chatty low-payload endpoints _(high)_

**Summary.** avg body=16B vs ~200B headers => headers are 12.6× the payload.

- Mean response body on `/executor/heartbeat` + `/executor/poll`: 16B (sample N=200).
- Conservative header overhead estimate: 200B/request.
- Bytes-on-the-wire is dominated by framing, not content.

**Mitigation.** Batch heartbeats: one POST /executor/heartbeats per supervisor covering N decks, body=[{deck_id,…}…]. Or move both loops to HTTP/2 multiplexed onto a single keep-alive connection, which amortizes the header cost across a longer-lived stream.

**Backing metrics:**
```json

  "header_to_body_ratio": 12.554927809165097,
  "n_requests": 200
}
```

### S7 — Write-lock contention proxy: `/executor/events` p99 vs N _(high)_

**Summary.** p99 grew 434.9× from N=10 to N=5000.

- Single SQLite write conn => commits serialize.
| N | p50 ms | p99 ms |
|---|---|---|
| 10 | 0.4 | 0.7 |
| 100 | 0.3 | 1.1 |
| 1000 | 3.5 | 16.1 |
| 5000 | 8.8 | 283.9 |

- Linear-in-N p99 = healthy. Super-linear = write queue saturating.

**Mitigation.** (a) Move events_log and heartbeat scalar updates onto a background writer goroutine fed by a buffered channel; (b) Replace single conn with a multi-writer DB (Postgres) if fleet exceeds 2–5k decks.

### C8 — Run-detail global+scoped poll redundancy (multi-tab scenario) _(med)_

**Summary.** global=1113.4 KiB, scoped=899.7 KiB; ~359.9 KiB estimated overlap.

- Multi-tab scenario captured two run-detail tabs polling both `/api/state` and `/api/runs/{id}/state` concurrently.
- Same events for those run_ids ship through both routes.
- Heuristic 40% overlap estimate on the scoped traffic: ~359.9 KiB.

**Mitigation.** On the client, pause the global `/api/state` poll when a run-detail tab is the only consumer of the page; or on the server, accept a `?exclude_run_ids=` parameter the FE sets from its open-tab registry.

**Backing metrics:**

### S10 — Heartbeats carry near-constant payloads (no-op rate) _(med)_

**Summary.** mean unique body-size per deck: 2.0; N=83 heartbeats.

- Across the capture window, decks heartbeat with body sizes that
  rarely change (`current_attempt_id` either stays NULL or a single
  UUID for the duration of one job). Body is essentially constant
  between transitions.
- Every heartbeat pays full TLS / TCP / HTTP framing cost for ~5 bytes
  of changing data.

**Mitigation.** Client-side suppression: skip the heartbeat if `(endpoint_url, current_attempt_id)` matches the last successfully-acked send AND the orchestrator's response said `last_seen <= max_silence/2`. Falls back to send on doubt.

**Backing metrics:**
```json

  "n_heartbeats": 83
}
```

### S11 — N+1 SQL pattern in bootstrap handlers _(med)_

**Summary.** 2 bootstrap handlers loop with per-row sub-queries.

- `handlers/state.go` contains: `bootstrapDecks`, `bootstrapRuns`.
- Each iteration of the bootstrap loop calls a `rowToX` helper that
  performs sub-queries (e.g. `LatestAttemptForJob`). At fleet size 100,
  one bootstrap = O(100) round-trips through the read pool.

**Mitigation.** Replace the per-row sub-queries with a single JOIN'd query per resource type. sqlc's `:many` queries make this concise.

**Backing metrics:**
```json

  "n_plus_1_handlers": [
    "bootstrapDecks",
    "bootstrapRuns"
  ]
}
```

### S8 — No gzip/br compression middleware on the orchestrator _(med)_

**Summary.** orchestrator does not advertise or emit Content-Encoding; would save ~0.0 KiB across 0 ≥4 KiB /api/state responses.

- Grep for `mw_compress*.go` / `Content-Encoding` in `backend/internal/api/`: no match.
- `/api/state` responses ≥ 4 KiB in capture: N=0 totaling 0.0 KiB.
- JSON compresses ~5× with gzip; ~80% wire savings on those responses.

**Mitigation.** Add a `Logging`-style middleware that wraps the ResponseWriter with `gzip.NewWriter` when the request has `Accept-Encoding: gzip`. One file, ~40 LOC. Zero impact on bootstraps under 4 KiB.

**Backing metrics:**
```json

  "big_state_responses": 0,
  "estimated_bytes_saved": 0
}
```

### S9 — X-Request-Id not propagated from executor outbox to orchestrator _(med)_

**Summary.** executor outgoing requests lack X-Request-Id header.

- Scanned `backend/internal/executor/outbox/*.go`: no `X-Request-Id` header set.
- Orchestrator middleware *generates* a request_id on inbound, but without executor-side propagation we can't trace an event from an executor's outbox commit through to the orchestrator's audit log.

**Mitigation.** In the outbox's http.Request build, set `req.Header.Set("X-Request-Id", attemptID + '-' + kind)`. Orchestrator's request-id middleware should prefer the inbound header if present (instead of overwriting).

**Backing metrics:**
```json
{
  "executor_outbox_propagates_request_id": false
}
```

### C3 — Bootstrap-on-open ships full snapshot each new tab _(low)_

**Summary.** first /api/state response: 0.00 KiB.

- Each new tab issues `GET /api/state?since_seq=0`. Bootstrap response in steady-state run: 0.00 KiB.
- At 100 decks the snapshot scales linearly with fleet size + recent runs.
- A team of operators opening 5 tabs a day pays this cost 5×.

**Mitigation.** Cache the snapshot in `localStorage` keyed on the orchestrator's `server_seq`; new tabs hydrate from local + only fetch the delta. Pair with a `BroadcastChannel` to invalidate across tabs.

### C7 — useQuery() calls with real queryFn (deviation from live-cache design) _(low)_

**Summary.** 5 `useQuery` call sites declare a non-throw queryFn.

- README claims the live `/api/state` poll replaces all per-resource refetches.
- These call sites declare an actual fetcher — each is a regression / intentional bypass that bypasses the live cache:
  - frontend/src/lib/live/use-supervisor-state.ts: queryFn=listSupervisorProcesses
  - frontend/src/lib/api/query-config.ts: queryFn=* cacheOnlyQueryFn
  - frontend/src/lib/api/index.ts: queryFn=cacheOnlyQueryFn
  - frontend/src/components/console/DeckChaosModal.tsx: queryFn=() => getDeckChaos(deckId)
  - frontend/src/components/console/DeckCard.tsx: queryFn=() => getDeckChaos(deck.id)

**Mitigation.** Audit each: if the data is in the live cache, switch to `queryFn: throw` + cache-read; if not (e.g., supervisor data), document the exception.

**Backing metrics:**
```json
{
  "n_real_queryfns": 5
}
```

### S5 — Error envelopes for state-mismatch errors embed the full Run _(low)_

**Summary.** 4 error variants ship `current_state: Run`; mean 4xx body in capture: 0.00 KiB.

- Variants found in `api/openapi.yaml`: VersionMismatchError, DuplicateResourceError, AlreadyTerminalError, InvalidTransitionError.
- Per `errors.go`, the orchestrator re-fetches the run, marshals it, and embeds it under `error.details.current_state`. Every version-off-by-one trip costs ≈ the full run summary in bytes.
- Captured 4xx responses: N=0 mean body=0.00 KiB.

**Mitigation.** Ship `details.current_version` only. Clients that need the new shape already re-fetch via the live cache on the next 1Hz tick.

**Backing metrics:**
```json
{
  "variants_embedding_run": 4,
  "avg_4xx_body_bytes": 0.0,
  "n_4xx": 0
}
```
